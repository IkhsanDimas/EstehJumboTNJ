1 Tambah ke Keranjang

User memilih produk, ukuran (Reguler/Jumbo), level es, level gula, dan topping opsional. Data dikirim via AJAX POST ke /cart/add. Sistem menghitung harga: harga dasar + modifier ukuran + topping. Item disimpan di session dengan key unik berdasarkan kombinasi opsi.

CartController.php → Session

2 Review Keranjang

User bisa menambah/mengurangi quantity, menghapus item, dan melihat subtotal. Data keranjang dibaca dari session dan ditampilkan di halaman /cart. Badge di navbar menunjukkan jumlah item secara real-time.

cart/index.blade.php → Session

3 Isi Form Checkout

User mengisi: nama, tipe pengiriman (delivery/pickup), alamat + pinpoint Google Maps, jadwal (sekarang/nanti), dan metode pembayaran (COD/transfer). Semua data divalidasi di server sebelum diproses.

checkout/index.blade.php → Validasi Server

4 Simpan ke Database

Data pesanan disimpan secara permanen ke tabel orders dan tabel order_items dalam satu database transaction. Setiap pesanan mendapat nomor unik format ETJ-XXXXXXXX.

CheckoutController.php → PostgreSQL (Supabase)

5 Generate Pesan WhatsApp

Sistem otomatis membuat pesan WhatsApp berisi: daftar item, opsi, quantity, alamat + link Maps, subtotal, ongkir, total, order ID, dan link tracking pesanan. URL dibuat dalam format wa.me/nomor?text=pesan.

CheckoutController.php → wa.me API

6 Konfirmasi & WhatsApp

Keranjang dikosongkan, user diarahkan ke halaman sukses. WhatsApp otomatis terbuka setelah 2 detik dengan pesan yang sudah terformat. Tersedia tombol manual dan link tracking pesanan.

checkout/success.blade.php → WhatsApp

💾 Di Mana Data Disimpan?

Data Penyimpanan Tabel Database Sifat
Keranjang (Cart) Session → Database sessions Sementara — hilang saat session expired atau checkout selesai
Pesanan (Order) Database orders Permanen — tersimpan selamanya, bisa dilihat di admin panel
Detail Item Pesanan Database order_items Permanen — setiap item dalam pesanan disimpan terpisah
Produk & Menu Database products Permanen — dikelola melalui admin panel
Kategori Database categories Permanen — Teh, Kopi, Ekstra Topping, dll
Akun Admin Database users Permanen — login admin panel
Pengaturan Toko Database store_settings Permanen — jam buka, ongkir, nomor WA

ℹ️ Catatan Penting: Meskipun keranjang menggunakan mekanisme "session", session driver diset ke database (bukan file/cookie). Artinya data session juga tersimpan di database PostgreSQL pada tabel sessions. Jadi semua data — termasuk keranjang sementara — sudah ada di database.
